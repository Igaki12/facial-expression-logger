# 2026-04-10 修正プラン

このプランは、4月9日のミーティング内容とスマートフォン利用時の操作性改善を、現行アプリの実装方針に合わせて整理したものです。

## 前提

- 現行の研究フローは `趣味 → 仕事` の段階型導線として維持する。
- 保存対象は、顔特徴量の数値データに加えて、フェーズごとの音声録音も含める。
- 映像ファイルと静止画は保存しない。
- 録画中・録音中を UI に併記し、カメラとマイクを使うことが分かる導線にする。
- Canvas / Video の親要素追従レスポンシブ化は行わない。
- MediaPipe Face Landmarker の生出力データは削らない。
- GitHub Pages 配信前提を壊さない。
- 変更後は `npm run build` を通し、`docs/` 出力が維持されることを確認する。

## 1. 音声録音と Praat 解析に向けた方針

- `getUserMedia` は `audio: true` に変更し、カメラとマイクを同時に取得する。
- `phase_recording` の自動開始・自動停止に合わせ、MediaRecorder で音声を録音する。
- 録音形式は追加依存なしで扱いやすい `audio/webm` を標準にする。
- `audio/webm` が使えないブラウザでは、`MediaRecorder.isTypeSupported()` で利用可能な MIME type にフォールバックする。
- 音声チャンクは React state に積まず、`useRef` に保持し、フェーズ終了時に IndexedDB へ保存する。
- 録画・録音の手動開始 / 停止ボタンは常設しない。
- ZIP 一括出力と WAV 変換は今回の実装対象に含めない。

## 2. 保存モデル

- IndexedDB スキーマは version `4` に上げる。
- 新規ストア `audioClips` を追加する。
- `audioClips` のキーは `["experimentId", "phaseKey"]` とする。
- `audioClips` には `by-experiment` index を作る。
- `AudioClipRecord` は以下を持つ。
  - `experimentId`
  - `phaseKey`
  - `label`
  - `startedAt`
  - `endedAt`
  - `mimeType`
  - `sizeBytes`
  - `audioBlob`
- 既存の `experiments` / `phases` / `frames` は維持する。
- 旧 `sessions` ストア削除と旧キー構造の `frames` / `phases` 再作成処理は残す。
- experiment 全体の削除時は、顔データと音声データをまとめて削除する。
- 音声だけの削除は、該当フェーズの `audioClips` だけを削除し、顔特徴量データには触れない。

## 3. 出力と履歴

- JSON 出力は既存の顔特徴量データ中心の形式を維持する。
- 音声は WebM ファイルとして個別出力する。
- ファイル名は JSON と対応づけやすいよう、`experiment_<startedAt>_<phaseKey>.webm` を基本にする。MIME type が `audio/mp4` にフォールバックした場合は `.m4a` にする。
- 履歴画面では、既存の顔特徴量の `確認` / `JSON` / `削除` を維持する。
- 履歴画面に、フェーズ別の音声行を追加する。
- 各音声行には `確認`、`出力`、`削除` を出す。
- `確認` は履歴画面内の `<audio controls>` で再生確認できる UI にする。
- 音声の確認は別ページにせず、モーダル必須にもせず、履歴画面内で完結させる。

## 4. スマートフォン向け UI / UX

- Video / Canvas の寸法制御は現行方式を維持する。
- ライブ映像に顔メッシュ描画用 Canvas を重ねない。
- 操作ボタンは、動画上に重ねるか、画面下部に固定する形で近接配置する。
- `stage-overlay` は動画下部の短い状態案内として維持する。
- `floating-prompt` は本文カード化せず固定表示を維持する。
- 文章は説明調を減らし、短く自然な文に差し替える。
- 例: `顔が見えています。このままで大丈夫です。` は `この位置でOKです。` のように短くする。
- モバイル幅ではタップ領域を最低 44px 程度確保する。
- 固定ボタンを使う場合は、本文末尾が隠れないように `padding-bottom` を調整する。

## 5. アイコン利用

- Iconify の `material-symbols` 系 SVG URL を使い、ボタンや履歴操作の視認性を補助する。
- アイコンは文字や絵文字ではなく、明確にアイコンとして見えるものにする。
- ボタン内のアイコンは `aria-hidden="true"` にする。
- 操作の意味はテキストまたは `aria-label` で保証する。
- 節目画面の大きめフラットアイコンは、現行どおり `work_transition` / `completion` / `debrief` で利用してよい。

## 6. 実装時に変更しないもの

- ライブ映像への顔メッシュ Canvas 重ね描き
- `video` / `canvas` の親要素幅・高さへの自動追従方式
- `frames` の保存を React state に積まない設計
- 履歴の顔特徴量プレビュー機能
- GitHub Pages 向けの `base: "/facial-expression-logger/"` と `docs/` 出力
